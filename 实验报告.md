# 实验报告

本报告融合 README、shiyan、数据集介绍、实验记录、研究报告、EXPERIMENT_STATE、AGENT_RUNBOOK、manifest 与 leaderboard 的当前记录，作为 S0 到 S7 设计入口的统一实验报告。

## 1. 项目目标

本项目围绕 DroneVehicle RGB-IR 无人机暗弱小目标检测展开，重点关注 dark-small、tiny 和 low-contrast 目标。当前主线以 YOLO11n 多尺度 RGB-IR 特征融合 E6 为基线，后续阶段围绕小目标增强、门控、损失修正、误报诊断、背景抑制和候选校准逐步收口。

## 6. 当前最终结论

1. S6.5 audit-only 收口完成，暂无有效新候选；S7 不直接进入论文最终验证，而是先拆成 `S7-A val-only 架构候选孵化` 和 `S7-B 论文完整验证` 两段。
2. E6 是当前唯一稳定主线基线，综合 mAP、dark-small AP、object-level AP 和误报水平最好。
3. E13_3b 单 seed 曾同时改善 AP 与 FP，但 corrected multi-seed 与 object-level gate 下未形成稳定候选。
4. E25_1_full、E26_1_full 和 E27_1_full 都能降低 FP/image 与 FPPI_dark，但 dark-small/tiny/low-contrast object-level AP 下降明显，因此 full 版结论均为 not_candidate。
5. E26_2a/E26_2b 因 class_confusion 源 CSV 的 split 字段为空，train-only 来源未被证明，标记为 failed_train_split_source_unverified，仅保留为失败审计样本。
6. 当前 `S7-A` 推荐顺序是 `S7_0 freeze & audit refresh -> S7_1 UTAH-lite quality-aligned head -> S7_3 Evidential reliability fusion lite -> S7_4 offset alignment lite`；`S7_2 ranking loss` 和 `S7_5 Frequency/Retinex shallow branch` 放在其后。
7. `S7-A` 的首个真正实验固定为 `S7_1: E6 + UTAH-lite quality-aligned head`，不是强模型 `E15`，也不是 test `E21`。

## 8. 2026-05-17 早期审查摘录

2026-05-17 的独立 review 已并入本报告，不再单独维护。该记录主要审查 E0-E8 早期链路：E0 固定 train-only 阈值和派生子集，E1/E2 使用官方 train 训练，E3/E4/E7_4 只在 val 上做后融合评估，E7_1/E7_2 使用同一 IR 协议做高分辨率对照。当时的核心判断是 E2 仍是暗弱主基线，E7_1/E7_2 高分辨率没有超过 E2，E7_4/E7_5 WBF 低于 IR-only 主指标。

后续 E8_2 已完成统一验证：Recall_small/Recall_dark 有提升，但 AP_dark-small=0.499926 未超过 E2=0.510692，因此暂停 E8_3/E8_4；E7_5 低于 E7_4，E7_6 与 E4 等权 WBF 口径重复而跳过。以上早期结论已被当前 S3-S6.5 的阶段表、leaderboard 和最终结论覆盖，仅作为历史背景保留。

## 9. 目录说明整合

原分散在各一级目录下的说明文件已整合到本节，后续只维护本实验报告和根目录 README。

### data

- 用途：存放原始数据和原始压缩包，只作为数据源，不直接作为训练输入。
- 主要内容：`data/DroneVehicle/raw/`、`data/DroneVehicle/zips/`、`data/VisDrone2019/`。
- 规范：不在 `data/` 内生成训练结果，不手工改动原始数据，不重新打乱或覆盖 DroneVehicle 官方 train/val/test 原始划分；训练使用 `prepared/` 中转换后的 YOLO 格式数据。

### prepared

- 用途：存放由原始数据转换得到的可训练数据和派生评测协议，不直接手工修改其中图像和标签。
- 主要内容：`prepared/dronevehicle_resplit/` 保留官方 split 与派生子集，`prepared/visdrone_yolo/` 作为辅助数据或横向参考。
- 协议：DroneVehicle-DarkSmall 不重新随机切分，只在官方 train/val/test 基础上做配对审计、裁边、AABB 转换、train-only 阈值统计和专项子集派生。

### configs

- 用途：存放数据集配置、子集配置和实验矩阵配置，训练脚本读取这里的配置，不在这里写结果。
- 主要内容：`configs/dronevehicle_resplit/`、`configs/dronevehicle_resplit/subsets/`、`configs/experiments/dark_small_next.yaml`、`configs/visdrone.yaml`。
- 规范：新实验优先通过 manifest 增加条目；实验输出和日志按阶段写入 `results/S*_*/` 对应目录；test set 只用于最终锁定评测。

### scripts

- 用途：存放数据处理、训练、验证、融合评估、批量实验和结果汇总脚本。
- 主要内容：`dataset_resplit_dronevehicle.py`、`dark_small_experiment_runner.py`、E1-E6 单实验脚本、关键图重建脚本、清理脚本和 `scripts/train/` 启动脚本。
- 规范：新实验优先走 `dark_small_experiment_runner.py`；shell wrapper 统一放到 `scripts/train/`；不在 `scripts/` 下保存训练产物。

### weights

- 用途：存放项目运行会引用的权重文件。
- 主要内容：`weights/pretrained/yolo11n.pt`；`weights/trained/` 当前不使用。
- 规范：训练得到的实验权重随阶段实验目录保存到 `results/S*_*/<experiment>/weights/`；固定评估权重至少保留 `best.pt`，需要断点续训的实验同时保留 `last.pt`。

### results

实验核心总结与下一步方向
